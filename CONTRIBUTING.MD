# Contributing

Thanks for your interest in contributing.

To keep incoming work reviewable and aligned with the project direction, please follow these rules:

- Raise an issue before starting work on a bug or feature request. The only exceptions are obvious critical bugs that need an immediate action and translations.
- Only issues labeled `accepted` are open for contribution. If an issue does not have that label, do not start work on it yet. I don't assign anyone to issues, unless I want to work on a specific task.
- One issue per pull request.
- LLM-generated code is allowed, BUT you are expected to review the code carefully and test all changes manually before submitting them.
- Do not add Claude, Codex, or any other agent as a co-author. You are fully responsible for every line of code you submit.
- `good first issue` doesn't mean that anyone here will teach you how to do Android development. It's a good first issue for newcomers to the project, not for people who have no idea about programming. Project uses straightforward Android configuration taught in every Android tutorial.

Following these rules makes triage, review, and maintenance much easier.
